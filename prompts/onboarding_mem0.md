# Промт онбординга: Mem0

> **Подход:** Векторная память Mem0 (Qdrant) через MCP
> **Для воспроизведения:** Cursor / Claude Code с Mem0 MCP сервером

## Промт

```
Ты — AI-агент, которого подключили к новому проекту. Твоя задача — провести онбординг:
разобраться в архитектуре, тестах и текущем состоянии проекта, чтобы быть готовым
к выполнению задач по тестированию.

### Источник контекста

Информация о проекте сохранена в Mem0 — векторной памяти:

1. Начни с `AGENTS.md` в корне — краткий обзор проекта
2. Выполни поиск в Mem0 по ключевым запросам:
   - "архитектура проекта" / "project architecture"
   - "тестовое покрытие" / "test coverage"
   - "известные проблемы" / "known issues"
   - "паттерны тестирования" / "test patterns"
   - "доменные сущности" / "domain entities"
3. Для каждого результата оцени релевантность — Mem0 возвращает
   семантически похожие записи, но не все могут быть актуальными
4. Проверь дату записей если доступна — старые записи могут содержать
   устаревшие данные (например, количество тестов до масштабирования)

### Что нужно понять

После онбординга ты должен знать:
- Какие сущности есть в проекте и как они связаны
- Сколько тестов, какого типа, какое покрытие по слоям
- Какие есть известные проблемы и flaky тесты
- Какие пробелы в покрытии существуют
- Какие конвенции именования и паттерны используются

### Верификация

После изучения контекста:
1. Запусти тесты: `go test ./...` — убедись что все проходят
2. Проверь покрытие: `go test -cover ./...` — сравни с данными из Mem0
3. Если данные из Mem0 расходятся с реальностью — обнови запись в Mem0
4. Сохрани результат онбординга как новую запись в Mem0

### Формат ответа

Дай краткий отчёт (5-10 пунктов):
- Что за проект, какие сущности
- Сколько тестов, какое покрытие
- Какие проблемы нашёл (расхождения, устаревшие данные в памяти)
- Какие записи в Mem0 обновил / создал
- К каким задачам готов приступить
```

## Особенности подхода

- **Плюсы:** Семантический поиск (не нужно знать точное имя файла), персистентность между сессиями
- **Минусы:** Нет структуры — flat list записей, нет связей между фактами, проблема staleness (старые записи не удаляются автоматически), требует настройки Mem0 + Qdrant
- **Стоимость:** Бесплатно (self-hosted) или $23/мес (Mem0 Cloud)
- **Токены на онбординг:** ~10-18K (AGENTS.md + несколько search запросов + результаты)
