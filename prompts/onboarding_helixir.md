# Промт онбординга: Helixir

> **Подход:** Графовая память Helixir (FastThink + reasoning chains) через MCP
> **Для воспроизведения:** Cursor / Claude Code с Helixir MCP сервером

## Промт

```
Ты — AI-агент, которого подключили к новому проекту. Твоя задача — провести онбординг:
разобраться в архитектуре, тестах и текущем состоянии проекта, чтобы быть готовым
к выполнению задач по тестированию.

### Источник контекста

Информация о проекте хранится в Helixir — графовой памяти с reasoning chains:

1. Начни с `AGENTS.md` в корне — краткий обзор проекта
2. Вызови `search_memory` для получения сохранённого контекста:
   - "архитектура проекта bean brew"
   - "тестовое покрытие coverage"
   - "известные проблемы flaky"
   - "паттерны тестирования"
3. Используй `search_reasoning_chain` для поиска цепочек рассуждений —
   предыдущие агенты могли сохранить свой анализ и выводы
4. Используй `search_by_concept` для навигации по графу знаний —
   связи между сущностями, тестами и проблемами
5. Проверь `search_incomplete_thoughts` — могут быть незавершённые
   рассуждения от предыдущей сессии

### Что нужно понять

После онбординга ты должен знать:
- Какие сущности есть в проекте и как они связаны
- Сколько тестов, какого типа, какое покрытие по слоям
- Какие есть известные проблемы и flaky тесты
- Какие пробелы в покрытии существуют
- Какие конвенции именования и паттерны используются

### Верификация

После изучения контекста:
1. Запусти тесты: `go test ./...` — убедись что все проходят
2. Проверь покрытие: `go test -cover ./...` — сравни с данными из Helixir
3. Если данные из Helixir расходятся с реальностью — обнови через `update_memory`
4. Сохрани результат онбординга через `think_start` → `think_add` → `think_commit`

### Формат ответа

Дай краткий отчёт (5-10 пунктов):
- Что за проект, какие сущности
- Сколько тестов, какое покрытие
- Какие проблемы нашёл (расхождения, устаревшие данные в памяти)
- Какие записи в Helixir обновил / создал
- К каким задачам готов приступить
```

## Особенности подхода

- **Плюсы:** Графовые связи между фактами, reasoning chains (цепочки рассуждений), поиск по концептам, незавершённые мысли — богатейший инструментарий для AI
- **Минусы:** Сложная настройка, самый дорогой по токенам (много API вызовов к MCP), данные могут устареть если не обновлять, инструмент молодой и менее стабильный
- **Стоимость:** Бесплатно (self-hosted)
- **Токены на онбординг:** ~20-35K (AGENTS.md + множественные search + reasoning chains + сохранение)
