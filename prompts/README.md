# Промты онбординга — Heisenbug 2026

Эта папка содержит промты для онбординга AI-агента в проект Bean & Brew
через 4 разных подхода к управлению контекстом.

## Файлы

### Базовые промты (v1)

| Файл | Подход | Источник контекста |
|------|--------|--------------------|
| `onboarding_md_files.md` | MD Files | Статичные MD файлы в репозитории |
| `onboarding_github_issues.md` | GitHub Issues | Тикеты с комментариями и чекбоксами |
| `onboarding_mem0.md` | Mem0 | Векторная память (Qdrant) через MCP |
| `onboarding_helixir.md` | Helixir | Графовая память с reasoning chains через MCP |

### Advanced промты (v3)

Базовые давали похожие результаты у всех подходов. v2-версии (с сигнатурными
задачами и 3-блоковым отчётом) дали машинный output: таблицы со скорами и
memory_id, скрытые человеческие наблюдения. v3 переписан под живого читателя.

| Файл | Что добавляет поверх MD |
|------|--------------------------|
| `adv_onboarding_md_files.md` | согласованность файлов, дока vs факт по покрытию |
| `adv_onboarding_github_issues.md` | прогресс/комментарии/living docs из тикетов |
| `adv_onboarding_mem0.md` | кросс-сессионные факты, tradeoffs, исторический контекст |
| `adv_onboarding_helixir.md` | ПОЧЕМУ команда сделала так, а не иначе — причинные связи |

Структура v3-промта: `<role>` (отвечать как человек) + `<task>` +
`<approach_capabilities>` (короткий pointer на соответствующий
`.cursor/rules/*.mdc` за tool-specific деталями) + `<context_sources>`
(MD для всех + свой источник) + `<execution_protocol>` (Phase 0–8 с
обязательными timestamp'ами в первой и последней строке отчёта) +
`<what_to_surface>` (конкретные примеры хороших наблюдений + явный
запрет на ID/score-таблицы) + `<writes_policy>` + `<reply_discipline>`
(дисциплина первой/последней строки ответа, строгий формат времени) +
`<output_format>` (5 секций: что я понял / покрытие дока vs факт / что
подход добавил поверх MD / найденные неточности и ограничения подхода /
список задач).

Описания MCP-инструментов (сигнатуры, параметры, ограничения) вынесены
в `.cursor/rules/*.mdc` — промты содержат только задачу онбординга.

## Как использовать

1. Выберите подход, который хотите протестировать
2. Скопируйте промт из блока ``` ``` в чат с AI-агентом (Cursor, Claude Code и т.д.)
3. Убедитесь что необходимые MCP-серверы подключены (для Mem0 / Helixir / GitHub)
4. Сравните результаты онбординга между подходами

## Структура промта

Все 4 промта имеют одинаковую структуру:
1. **Задача** — одинаковая: онбординг проекта
2. **Источник контекста** — различается: где брать информацию
3. **Что нужно понять** — одинаковое: 5 пунктов
4. **Верификация** — одинаковая база + специфичные действия для подхода
5. **Формат ответа** — одинаковый: краткий отчёт
