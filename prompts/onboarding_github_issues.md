# Промт онбординга: GitHub Issues

> **Подход:** GitHub Issues как источник контекста через MCP
> **Для воспроизведения:** Cursor с GitHub MCP / Claude Code с `gh` CLI

## Промт

```
Ты — AI-агент, которого подключили к новому проекту. Твоя задача — провести онбординг:
разобраться в архитектуре, тестах и текущем состоянии проекта, чтобы быть готовым
к выполнению задач по тестированию.

### Источник контекста

Информация о проекте хранится в GitHub Issues репозитория:

1. Начни с `AGENTS.md` в корне — краткий обзор проекта
2. Загрузи список GitHub Issues: открытые и закрытые
3. Для каждого тикета изучи:
   - Тело тикета (описание задачи, acceptance criteria, чекбоксы)
   - Комментарии (ход работы, результаты, метрики)
   - Дату создания и последнего обновления
   - Метки (test:unit, test:integration, context:coverage и т.д.)
4. Обрати внимание на:
   - Закрытые тикеты — что уже сделано
   - Открытые тикеты — что ещё предстоит
   - Чекбоксы — какие пункты выполнены, какие нет
   - Даты последних комментариев — насколько актуальна информация
5. Изучи правила работы с тикетами в `.cursor/rules/github.mdc`

### Что нужно понять

После онбординга ты должен знать:
- Какие сущности есть в проекте и как они связаны
- Сколько тестов, какого типа, какое покрытие по слоям
- Какие есть известные проблемы и flaky тесты
- Какие пробелы в покрытии существуют
- Какие конвенции именования и паттерны используются

### Верификация

После изучения контекста:
1. Запусти тесты: `go test ./...` — убедись что все проходят
2. Проверь покрытие: `go test -cover ./...` — сравни с данными в тикетах
3. Если данные в тикетах расходятся с реальностью — обнови комментарий в тикете
4. Если все чекбоксы в тикете выполнены, а тикет открыт — закрой его

### Формат ответа

Дай краткий отчёт (5-10 пунктов):
- Что за проект, какие сущности
- Сколько тестов, какое покрытие
- Какие проблемы нашёл (расхождения, устаревшие данные)
- Какие тикеты обновил / закрыл
- К каким задачам готов приступить
```

## Особенности подхода

- **Плюсы:** Динамический прогресс (чекбоксы, комменты), даты актуальности, двусторонняя связь (агент и обновляет и читает)
- **Минусы:** Требует MCP-подключение к GitHub, больше токенов на загрузку тикетов
- **Стоимость:** Бесплатно (GitHub Issues бесплатны)
- **Токены на онбординг:** ~15-25K (AGENTS.md + загрузка и чтение всех тикетов с комментариями)
